# Database Exports Directory

This directory contains exported database backups from the Tender Management System.

## 📁 File Types

### 1. Full SQL Dump
**Filename:** `tender-management-full-export-{timestamp}.sql`

Complete database backup including:
- Database schema (tables, indexes, constraints)
- All database functions
- All data as INSERT statements
- Ready to restore to a new database

**Usage:**
```bash
# Restore to Supabase (using psql)
psql -h db.ecvqhfbiwqmqgiqfxheu.supabase.co -U postgres -d postgres -f tender-management-full-export-{timestamp}.sql

# Or import via Supabase Dashboard
# 1. Go to Supabase Dashboard → SQL Editor
# 2. Copy contents of .sql file
# 3. Run the SQL
```

### 2. JSON Exports (per table)
Individual table exports in JSON format:
- `tender_companies-{timestamp}.json`
- `tender_users-{timestamp}.json`
- `tender_tenders-{timestamp}.json`
- `tender_tender_history-{timestamp}.json`

**Usage:**
```javascript
// Load and use in Node.js
import companiesData from './tender_companies-{timestamp}.json'
console.log(companiesData)

// Or use for data analysis, migration, etc.
```

### 3. Export Report
**Filename:** `export-report-{timestamp}.txt`

Human-readable summary containing:
- Export date and time
- Table row counts
- List of generated files
- Export statistics

## 🚀 How to Create New Export

Run the export command:
```bash
npm run export-db
```

This will:
1. Connect to your Supabase database
2. Fetch all data from all tables
3. Generate SQL dump and JSON files
4. Create a detailed report
5. Save everything in this directory

**Best Practices:**
1. **Never commit exports to Git** (already in .gitignore)

### Recommended Schedule:
- **Daily:** For production databases
- **Weekly:** For development databases
- **Before major changes:** Always backup first!

### Retention Policy:
- Keep last 7 daily backups
- Keep last 4 weekly backups
- Keep monthly backups for 1 year

## 🛠️ Troubleshooting

### Export fails with "Connection error"
- Check your `.env.local` file has correct Supabase credentials
- Verify network connection to Supabase

### Missing data in export
- Check table permissions in Supabase
- Verify RLS policies allow reading data
- Check Supabase service role key if using

### Large file sizes
- Consider compressing exports: `gzip tender-management-full-export-*.sql`
- Archive old exports to separate storage
- Clean up old history records if needed
